## UI COMPONENT ARCHITECTURE

1. JIRA REQUIREMENT SUMMARY

Story Description:
AS a parent or guardian I WANT to allocate and manage funds for my child’s youth account SO THAT I can teach financial responsibility while maintaining visibility and control over spending

Acceptance Criteria:
- Scenario 1 – Access youth account dashboard: Given the parent is logged into the banking web application When the parent navigates to the youth account section Then the system should display the youth account dashboard with current balance and recent activity
- Scenario 2 – Allocate funds to youth account: Given the parent is on the youth account dashboard When the parent selects “Add Funds” Then the system should allow the parent to transfer funds from their primary account to the youth account
- Scenario 3 – Set spending limit: Given the parent is viewing the youth account management section When the parent configures a weekly spending limit Then the system should save and display the limit on the dashboard
- Scenario 4 – View youth spending activity: Given the parent is viewing the youth account dashboard When the parent selects the activity section Then the system should display recent transactions made from the youth account
- Scenario 5 – Handle insufficient balance during fund allocation: Given the parent attempts to transfer funds When the selected parent account does not have sufficient balance Then the system should display an error message indicating insufficient funds

UI Tasks:
- Design youth account dashboard layout
- Design fund transfer interaction for youth account
- Design spending limit configuration interface
- Design youth account activity view

2. HTML STRUCTURE SUMMARY (SOURCE OF TRUTH)

Header:
- Contains search field, notification icon, and avatar/user chip for parent

Sidebar / Navigation:
- Navigation for: Dashboard, Add Funds, Set Limit, Activity
- Section for Errors & Confirmations (Dashboard Load Error, Insufficient Balance Error, Invalid Limit Error, Transfer Confirmation, Transfer Success, Limit Update Confirmation)
- Footer with version and copyright

Main Container:
- Dashboard page: Account balance card, spending limit card (with edit), recent activity card (with list), financial literacy tips card
- Add Funds page: Source account select, amount input, frequent templates, transfer limit badge, transfer/cancel buttons
- Transfer Confirmation modal: Review source, amount, recipient, confirm/back buttons
- Transfer Success modal: Success message, updated balance, dashboard/activity buttons
- Set Limit page: Current limit, new limit input, guidance, save/cancel buttons
- Limit Update Confirmation modal: Success message, dashboard button
- Activity page: Transaction list (date, merchant, amount, balance)
- Error modals: Snackbar-style error for dashboard load, insufficient balance, invalid limit

Sections:
- Cards (Account Balance, Spending Limit, Recent Activity, Tips)
- Forms (Add Funds, Set Limit)
- Lists (Recent Activity, Transaction List)
- Buttons (Add Funds, Set Limit, View Activity, Edit, Save, Cancel, Transfer, Confirm, Retry)
- Modals (Transfer Confirmation, Transfer Success, Limit Update Confirmation, Error Snackbars)

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data (balance, limit, recent activity)
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Add funds
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update limit
- GET /api/youth-accounts/{youthAccountId}/transactions → List transactions

9. API TO UI MAPPING

- DashboardPage → GET /api/youth-accounts/{youthAccountId}/dashboard

13. INTERACTION FLOWS

User Action → UI → API
- Click "Add Funds"
  → Open FundTransferForm
  → Submit
  → Show TransferConfirmationModal
  → Confirm
  → Call POST /api/youth-accounts/{youthAccountId}/fund-transfer
  → Show TransferSuccessModal

- Click "Set Limit"
  → Open SpendingLimitPage
  → Enter new limit
  → Save
  → Call PUT /api/youth-accounts/{youthAccountId}/spending-limit
  → Show LimitUpdateConfirmationModal

- Click "View Activity"
  → Open ActivityPage
  → Fetch transactions via GET /api/youth-accounts/{youthAccountId}/transactions

- Error (e.g., insufficient balance)
  → Show InsufficientBalanceSnackbar
