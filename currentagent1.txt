UI Component Implementation Specifications

1. Youth Account Dashboard

Component Name
YouthAccountDashboardPage

Purpose
Displays youth account overview, balance, spending limit, recent activity, and quick actions for fund transfer, limit setting, and activity viewing.

Dependencies
- SidebarNavigation
- HeaderBar
- Card
- Button
- Avatar
- Chip-Tag
- Tabs
- Snackbar
- ListItem

Library Components Used
- aava-default-card
- aava-card-header
- aava-card-content
- aava-card-footer
- aava-button
- aava-avatar
- aava-chip-tag
- aava-tabs
- aava-snackbar

TypeScript Specification
Inputs:
- youthAccountId: string
- dashboardData: YouthAccountDashboard
Outputs:
- onAddFunds: {youthAccountId}
- onSetLimit: {youthAccountId}
- onViewActivity: {youthAccountId}
State:
- loading: boolean
- error: string | null
- dashboardData: YouthAccountDashboard
Methods:
- fetchDashboard() → Loads dashboard data from API
- handleAddFunds() → Navigates to fund transfer
- handleSetLimit() → Navigates to limit configuration
- handleViewActivity() → Navigates to activity view

HTML Structure
- SidebarNavigation (Youth Account, Errors & Confirmations)
- HeaderBar (Search, Notifications, Avatar)
- MainContainer
  - Card: Account Balance
  - Card: Spending Limit (with Edit button)
  - Card: Recent Activity (list, View All button)
  - Card: Financial Literacy Tips
  - Snackbar: Error messages

CSS Specification
Layout Styles:
- Flex and grid for main container and cards
- Responsive sidebar and header
Component Styles:
- Card: border-radius, shadow, padding
- Button: variants, pill shape, hover/pressed effects
Responsive Rules:
- Sidebar collapses on mobile
- Cards stack vertically on small screens

Jira Requirement Summary
Story Description
AS a parent or guardian I WANT to allocate and manage funds for my child’s youth account SO THAT I can teach financial responsibility while maintaining visibility and control over spending

Acceptance Criteria
- Access youth account dashboard (balance, activity)
- Allocate funds to youth account
- Set spending limit
- View youth spending activity
- Handle insufficient balance during fund allocation

UI Tasks
- Design youth account dashboard layout
- Design fund transfer interaction
- Design spending limit configuration interface
- Design youth account activity view

HTML Structure Summary
Header
- Search, notifications, avatar
Navigation
- Sidebar with dashboard, transfer, limit, activity, errors, confirmations
Main Container
- Cards for balance, limit, activity, tips
- Modals for transfer, limit, confirmations, errors
Cards
- Account balance, spending limit, recent activity, tips
Forms
- Fund transfer, limit edit
Lists
- Transaction list
Buttons
- Add Funds, Set Limit, View Activity, Transfer, Save, Cancel, Retry

Reusable Components Identified
- Card
- Button
- Modal
- Avatar
- Chip-Tag
- Tabs
- Snackbar
- ListItem

API Summary
- GET /api/youth-accounts/{youthAccountId}/dashboard: Fetch dashboard data
- POST /api/youth-accounts/{youthAccountId}/fund-transfer: Transfer funds
- PUT /api/youth-accounts/{youthAccountId}/spending-limit: Set/update limit
- GET /api/youth-accounts/{youthAccountId}/transactions: List transactions

API to UI Mapping
- YouthAccountDashboardPage → GET dashboard
- FundTransferModal → POST fund-transfer
- SpendingLimitModal → PUT spending-limit
- YouthAccountActivityPage → GET transactions

API Compatibility
Compatible
- All required endpoints exist
- Request/response models match UI needs
Missing fields
- None
Required transformations
- UI must map account selection to sourceAccountId
- UI must handle error codes/messages for insufficient balance, invalid limit

Jira vs HTML Validation
Matching Elements:
- Dashboard, fund transfer, limit, activity, errors, confirmations
Missing Elements:
- None
Extra Elements:
- Financial literacy tips (not in Jira, but useful)

Data Flow / Props
ParentComponent (Dashboard)
→ ChildComponent (Card, Button, Modal, Snackbar)
FundTransferModal
→ Card, Select, Textbox, Badge, Button
SpendingLimitModal
→ Card, Textbox, Button
YouthAccountActivityPage
→ Card, ListItem, Tabs

Layout Structure
- SidebarNavigation (fixed left)
- HeaderBar (top)
- MainContainer (cards, modals)
- Responsive grid for cards

API Integration Strategy
- APIs called on page load, form submit
- Data stored in state per page/component
- Error handling via Snackbar/Modal

Component Definitions
- Card: Container for content, metadata, actions
- Button: Triggers actions, navigation
- Modal: Overlay for forms, confirmations
- Avatar: Parent identity
- Chip-Tag: Labels, status
- Tabs: Filter, navigation
- Snackbar: Error, confirmation messages
- ListItem: Transaction, activity display

Interaction Flows
- Parent logs in → Dashboard loads
- Add Funds → FundTransferModal opens → Transfer → Confirmation/Success/Error
- Set Limit → SpendingLimitModal opens → Save → Confirmation/Error
- View Activity → Activity page loads
- Errors handled via Snackbar/Modal

Performance suggestions
