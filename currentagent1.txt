Jira Requirement Summary

Story Description
AS a parent or guardian I WANT to allocate and manage funds for my child’s youth account SO THAT I can teach financial responsibility while maintaining visibility and control over spending

Acceptance Criteria
- Scenario 1: Access youth account dashboard (dashboard with balance, recent activity)
- Scenario 2: Allocate funds to youth account (add funds flow)
- Scenario 3: Set spending limit (limit config flow)
- Scenario 4: View youth spending activity (transaction list)
- Scenario 5: Handle insufficient balance during fund allocation (error dialog)

UI Tasks
- Design youth account dashboard layout
- Design fund transfer interaction for youth account
- Design spending limit configuration interface
- Design youth account activity view

HTML Structure Summary
- Sidebar (navigation)
- Header (search, notification, avatar)
- Main Container (dashboard, transfer, limit, activity, dialogs)
- Cards (balance, limit, activity, tips)
- Forms (transfer, set limit)
- Lists (transactions)
- Buttons (actions)

Reusable Components Identified
- Card
- Button
- Tabs
- Avatar
- Chip-Tag
- Snackbar
- Grid
- Sidebar
- Header
- Dropdown
- Dialog
- Badge

API Summary
- GET /api/youth-accounts/{youthAccountId}/dashboard → Dashboard data (balance, limit, recent activity)
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Transfer funds
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Set/update weekly limit
- GET /api/youth-accounts/{youthAccountId}/transactions → List transactions

API to UI Mapping
- YouthAccountDashboardPage → GET /dashboard
- FundTransferModal → POST /fund-transfer
- SetSpendingLimitModal → PUT /spending-limit
- TransactionListPage → GET /transactions
- Error/Success Dialogs → API error/success responses

API Compatibility
Compatible
- All required UI flows are supported by APIs
Missing fields
- None
Required transformations
- Map API models to UI props (e.g., dashboardData.accountBalance → AccountBalanceCard.balance)

Jira vs HTML Validation
Matching Elements:
- Dashboard, Add Funds, Set Limit, Activity, Error dialogs, Confirmation dialogs
Missing Elements:
- None
Extra Elements:
- Financial Tips Card (bonus, not in Jira but value add)

Layout Structure
- AppShell (SidebarNavigation + Main)
- Main (HeaderBar + Content)
- Content (PageHeader + Cards/Modals)

API Integration Strategy
- On page load, fetch dashboard data
- On Add Funds, open FundTransferModal, POST to /fund-transfer, show confirmation/error dialog
- On Set Limit, open SetSpendingLimitModal, PUT to /spending-limit, show confirmation/error dialog
- On View Activity, fetch transactions
- Error handling: show SnackbarAlert or DialogPopup
- State: Store dashboard data, transactions, limits in page state

Component Definitions
- YouthAccountDashboardPage: Orchestrates all dashboard UI and data
- SidebarNavigation: Handles navigation
- HeaderBar: Search, notifications, avatar
- AccountBalanceCard: Shows balance
- SpendingLimitCard: Shows/edit limit
- RecentActivityCard: Shows recent transactions
- FinancialTipsCard: Shows tips
- SnackbarAlert: Shows notifications
- FundTransferModal: Add funds flow
- SetSpendingLimitModal: Set limit flow
- TransactionListPage: All transactions
- DialogPopup: Confirmation, error, success dialogs

Interaction Flows
- Parent logs in → sees dashboard
- Click Add Funds → FundTransferModal → Confirm → POST /fund-transfer → Success/Error Dialog
- Click Set Limit → SetSpendingLimitModal → Save → PUT /spending-limit → Success/Error Dialog
- Click View Activity → TransactionListPage → GET /transactions
- Errors (insufficient balance, invalid limit, load error) → SnackbarAlert/DialogPopup

Improvements

- Add accessibility ARIA labels per component docs
Performance suggestions
