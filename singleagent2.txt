### 4.1 DashboardPage

Component Type:
Page

Mapped From A1:
DashboardPage

Purpose:
Top-level container for dashboard UI, orchestrates layout and data flow

Parent:
None

Children:
- HeaderComponent
- SidebarComponent
- DashboardMain
- AddFundsModal

Dependencies:
- HeaderComponent → layout
- SidebarComponent → navigation
- DashboardMain → feature
- AddFundsModal → modal

Library Components Used:
- Modal → modal display
- Button → actions

--------------------------------------------------
TypeScript Specification
--------------------------------------------------

Inputs:
- userId: string

Outputs:
- onAddFunds(amount: number)

State:
- showAddFundsModal: boolean
- dashboardData: DashboardData

Methods:
- handleAddFunds() → opens AddFundsModal
- fetchDashboardData() → loads dashboard data

--------------------------------------------------
HTML STRUCTURE (PSEUDO CODE ONLY)
--------------------------------------------------

<DashboardPageContainer>
  <HeaderComponent />
  <SidebarComponent />
  <DashboardMain dashboardData={dashboardData} />
  {showAddFundsModal && <AddFundsModal onSubmit={onAddFunds} />}
</DashboardPageContainer>

--------------------------------------------------
CSS SPECIFICATION
--------------------------------------------------

Layout:
- display: flex; flex-direction: row;
- Header: fixed top
- Sidebar: fixed left
- MainContent: flex-grow

Spacing:
- gap: 24px between sections
- padding: 32px main content

Component Styles:
- .dashboard-container
- .header
- .sidebar
- .main-content

Responsive:
- @media (max-width: 900px): sidebar collapses
- @media (max-width: 600px): header stacks

--------------------------------------------------
API INTEGRATION
--------------------------------------------------

- fetchDashboardData() → GET /api/dashboard → loads dashboard data
- onAddFunds() → POST /api/fund-transfer → submits fund transfer

--------------------------------------------------
ERROR HANDLING
--------------------------------------------------

- API failure → show snackbar
- Validation error → inline message

--------------------------------------------------
INTERACTION FLOW
--------------------------------------------------

User Action → UI → API
- Click 'Add Funds' → open AddFundsModal → submit → API call

--------------------------------------------------
