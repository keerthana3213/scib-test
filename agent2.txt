### 4.1 YouthAccountPage

Component Type:
Page

Mapped From A1:
YouthAccountPage

Purpose:
Top-level container for the youth account dashboard, orchestrates layout, data flow, and API integration.

Parent:
N/A

Children:
- HeaderComponent
- SidebarComponent
- DashboardSection
- FundTransferSection
- SpendingLimitSection
- ActivitySection
- ConfirmationModals
- ErrorSnackbars

Dependencies:
- All child components

Library Components Used:
- Grid (for layout)

TypeScript Specification
Inputs:
- user: object
- navItems: array
- dashboardData: object
- transactions: array
- tips: array
- error: string
- snackbarType: string
Outputs:
- onSearch(payload)
- onNavSelect(payload)
- onTransfer(payload)
- onSaveLimit(payload)
- onRetry()
State:
- selectedSection: string
- modalState: {transferOpen, limitOpen, confirmationOpen, successOpen}
- errorState: {dashboard, transfer, limit}
Methods:
- fetchDashboard() → Loads dashboard data
- handleTransfer() → Handles fund transfer
- handleLimitSave() → Handles limit update
- handleSnackbar() → Shows error messages

HTML STRUCTURE (PSEUDO CODE ONLY)
<YouthAccountPage>
  <HeaderComponent user onSearch />
  <Grid>
    <SidebarComponent navItems selected onNavSelect />
    <MainContainer>
      <DashboardSection />
      <FundTransferSection />
      <SpendingLimitSection />
      <ActivitySection />
      <ConfirmationModals />
      <ErrorSnackbars />
    </MainContainer>
  </Grid>
</YouthAccountPage>

CSS SPECIFICATION
Layout:
- display: grid (main)
- grid-template-columns: sidebar + main
- gap: 24px
Spacing:
- padding: 32px main content
Component Styles:
- .main-container, .dashboard-section, .modals
Responsive:
- @media (max-width: 1280px): sidebar collapses, grid stacks

API INTEGRATION
- fetchDashboard() → GET /api/youth-accounts/{id}/dashboard
- handleTransfer() → POST /api/youth-accounts/{id}/fund-transfer
- handleLimitSave() → PUT /api/youth-accounts/{id}/spending-limit
- fetchTransactions() → GET /api/youth-accounts/{id}/transactions

ERROR HANDLING
- API failure → show snackbar (DashboardLoadError, InsufficientBalanceError, InvalidLimitError)
- Validation error → inline message or snackbar

INTERACTION FLOW
User Action → UI → API
- Click nav → section change
- Click Add Funds → open modal → submit → API call
- Click Set Limit → open modal → submit → API call
- Click View Activity → show transaction list
- API error → show snackbar

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- All API error handling must use Snackbar
- All modals must use Modal from library
- All cards must use Card from library
- All navigation must use Sidebar from library
- All icons, avatars, and tabs must use library components
