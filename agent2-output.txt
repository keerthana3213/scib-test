    <SpendingLimitModal ... />
    <ActivityPage ... />
    <SnackbarAlert ... />
    <ErrorModals ... />
  </MainContentArea>

CSS SPECIFICATION

- Responsive: sidebar collapses on mobile, header stacks
Spacing:
- gap: 24px main, 16px cards
Component Styles:
- Main area: background #f8f9fa
Responsive:
- @media (max-width: 900px): sidebar overlays, main stacks

API INTEGRATION
- fetchDashboardData() → GET /api/youth-accounts/{youthAccountId}/dashboard
- fetchTransactions() → GET /api/youth-accounts/{youthAccountId}/transactions

ERROR HANDLING
- API failure → show ErrorModals or SnackbarAlert

INTERACTION FLOW
- Menu select → change main content
- Modal open/close → update state

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- All API error states must be handled with ErrorModals or SnackbarAlert
- Accessibility: All interactive elements must have aria-labels
- Performance: Use OnPush change detection, lazy load modals/pages
