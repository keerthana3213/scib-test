FundTransferModal

Library Components Used:
- Modal → container
- Dropdown → source account select
- Textbox → amount input
- Badge → transfer limit
- Button → transfer/cancel
- ChipTag → frequent templates

TypeScript Specification

Inputs:
- sourceAccounts: Account[]
- transferAmount: number
- transferLimit: number

Outputs:

- handleCancel(): void → close modal

  <ModalHeader title="Add Funds" />

    <Textbox value={amount} onChange={handleAmountChange} />
    <ChipTag label="Frequent Templates" />
    <Badge label={transferLimit} />
  </ModalContent>
  <ModalFooter>
    <Button label="Transfer" onClick={handleTransfer} />
    <Button label="Cancel" onClick={handleCancel} />
  </ModalFooter>
</Modal>

- .modal-content

Responsive:
- @media (max-width: 600px) → full screen modal

- handleTransfer() → POST /api/youth-accounts/{youthAccountId}/fund-transfer

ERROR HANDLING
- API failure → show SnackbarComponent with error message
- Insufficient balance → show error message

INTERACTION FLOW

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- All API errors must be handled via SnackbarComponent
- Responsive rules must be included for all main components
- All modals must be overlay, full screen on mobile
- Data flow must strictly follow parent → child mapping from A1
