### 4.1 YouthAccountPage

Component Type:
Page

Mapped From A1:
YouthAccountPage

Purpose:
Top-level container for the youth account management experience. Orchestrates layout, navigation, and data flow.

Parent:
N/A (root)

Children:
- HeaderComponent
- SidebarComponent
- DashboardPage
- FundTransferModal
- SpendingLimitModal
- ActivityPage
- ConfirmationModal
- SuccessModal
- ErrorSnackbar

Dependencies:
- Layout components
- Feature components
- Shared components

Library Components Used:
- Grid (for main layout)
- Divider (section separation)

TypeScript Specification

Inputs:
- user: { name: string, avatarUrl: string }
- notifications: Notification[]
- menuItems: MenuItem[]
- dashboardData: { balance, limit, recentActivity, tips }
- sourceAccounts: Account[]
- transferLimit: number
- templates: Template[]
- currentLimit: number
- guidance: string
- transactions: Transaction[]
- details: { source, amount, recipient }
- message: string
- updatedBalance?: number
- error: { message: string, type: 'error' | 'warning' }

Outputs:
- onMenuSelect(section: string)
- onAddFunds()
- onSetLimit()
- onViewActivity()
- onTransfer(payload)
- onSaveLimit(payload)
- onConfirm(payload)
- onCloseModal()
- onDismissError()

State:
- selectedSection: string
- modalState: { fundTransferOpen, setLimitOpen, confirmationOpen, successOpen, errorOpen }

Methods:
- handleMenuSelect(section)
- handleAddFunds()
- handleSetLimit()
- handleViewActivity()
- handleTransfer(payload)
- handleSaveLimit(payload)
- handleConfirm(payload)
- handleCloseModal()
- handleDismissError()

HTML STRUCTURE (PSEUDO CODE ONLY)

<GridContainer>
  <HeaderComponent user notifications />
  <SidebarComponent selectedSection menuItems onMenuSelect />
  <MainContentArea>
    <DashboardPage dashboardData onAddFunds onSetLimit onViewActivity />
    <FundTransferModal sourceAccounts transferLimit templates onTransfer onCancel />
    <SpendingLimitModal currentLimit guidance onSave onCancel />
    <ActivityPage transactions onFilter />
    <ConfirmationModal details onConfirm onBack />
    <SuccessModal message updatedBalance onClose />
    <ErrorSnackbar message type onDismiss />
  </MainContentArea>
</GridContainer>

CSS SPECIFICATION

Layout:
- display: grid; grid-template-areas: 'sidebar header' 'sidebar main';
- Sidebar: fixed width, full height
- Header: sticky top
- MainContentArea: flex column, gap between cards/sections

Spacing:
- padding: 24px main content
- gap: 16px between cards

Component Styles:
- Sidebar: background, active menu highlight
- Header: shadow, avatar alignment

Responsive:
- Sidebar collapses to icons on < 900px
- Main content stacks vertically on mobile

API INTEGRATION
- All child components handle their own API calls via props/events

ERROR HANDLING
- ErrorSnackbar shown for any API/UI error

INTERACTION FLOW
- Menu select → update selectedSection → show relevant page/modal
- Add Funds/Set Limit → open modal → submit → API call → confirmation/success/error

7. API USAGE SUMMARY

Component → API:
- DashboardPage → GET /api/youth-accounts/{youthAccountId}/dashboard
- FundTransferModal → POST /api/youth-accounts/{youthAccountId}/fund-transfer
- SpendingLimitModal → PUT /api/youth-accounts/{youthAccountId}/spending-limit
- ActivityPage, TransactionListCard → GET /api/youth-accounts/{youthAccountId}/transactions
- ConfirmationModal → POST/PUT as above
- ErrorSnackbar → Error responses from any API

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- All API errors must be surfaced via ErrorSnackbar
- All modals must be accessible and keyboard navigable
- Use OnPush change detection for all feature components
- Lazy load modals and activity page for performance
