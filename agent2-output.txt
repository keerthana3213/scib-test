### 4.5 ActionButton

Component Type: Shared

Mapped From A1: ActionButtons (aava-button)

Purpose: Trigger user actions and workflows with consistent styling and behavior

Parent: Various components (Dashboard, Forms, Modals)

Children: None (atomic component)

Dependencies:
- @aava/play-core (AavaButtonComponent)
- @aava/play-core (AavaIconComponent)

Library Components Used:
- aava-button → Main button component
- aava-icon → Button icons

TypeScript Specification

Inputs:
- label: string
- variant: 'primary' | 'secondary' | 'success' | 'warning' | 'danger'
- size: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
- disabled: boolean
- loading: boolean
- iconName?: string
- iconPosition?: 'left' | 'right' | 'only'
- actionType?: string
- permissions?: string[]

Outputs:
- onClick: () => void
- onActionComplete: (result: any) => void
- onActionError: (error: any) => void

State:
- isProcessing: boolean
- actionResult: any
- error: string | null

Methods:
- handleClick() → Process button click and execute action
- executeAction() → Execute the specific action type
- validatePermissions() → Check if user has required permissions
- showFeedback(type: string, message: string) → Show user feedback

HTML STRUCTURE (PSEUDO CODE ONLY)

<aava-button
  [label]="label"

  [processing]="isProcessing"
  [iconName]="iconName"
  [iconPosition]="iconPosition"
  [iconColor]="getIconColor()"
  (userClick)="handleClick()"
  [customStyles]="customButtonStyles"
  class="action-button"
  [class]="'action-' + actionType">
</aava-button>

CSS SPECIFICATION

Layout:
- display: inline-flex
- align-items: center
- justify-content: center

Spacing:
- margin: 8px (when in button groups)
- padding: inherited from aava-button

Component Styles:
- .action-button: base styles for all action buttons
- .action-primary: primary action styling
- .action-secondary: secondary action styling
- .action-danger: destructive action styling
- hover and focus states inherited from aava-button

Responsive:
- @media (max-width: 768px): adjust size to 'sm' for mobile
- stack vertically in button groups on mobile

API INTEGRATION

- executeAction() → POST /actions/{actionType} → Execute specific action
- validateAction() → GET /actions/{actionType}/validate → Validate action before execution

ERROR HANDLING

- API failure → show error snackbar with retry option
- Permission denied → disable button and show tooltip
- Network timeout → show timeout message and retry option
- Validation error → show inline error message

INTERACTION FLOW

User Action → UI → API
- Click action button → show loading state → call POST /actions/{actionType} → show success/error feedback → refresh relevant data
- Hover disabled button → show tooltip explaining why disabled → no API call
- Long press (mobile) → show action confirmation → execute on confirm
