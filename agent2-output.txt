### 4.1 YouthAccountPage

Component Type:
Page

Mapped From A1:
YouthAccountPage

Purpose:
Top-level container for the youth account dashboard and flows. Handles layout, routing, and data orchestration.

Parent:
N/A (root)

Children:
- HeaderComponent
- SidebarComponent
- MainContainer (DashboardPage, FundTransferModal, SpendingLimitModal, ActivityPage, ConfirmationModal, SnackbarComponent)

Dependencies:
- React Router (for page switching)
- Context/State management

Library Components Used:
- None directly (delegates to children)

TypeScript Specification
Inputs:
- user: { name: string, avatarUrl: string }
- dashboardData: object
- transactions: Transaction[]
- sourceAccounts: Account[]
- currentLimit: number
- tips: string[]
Outputs:
- onSearch(query: string)
- onNavigate(page: string)
- onTransfer(details)
- onSaveLimit(newLimit: number)
- onSnackbarDismiss()
State:
- selectedPage: string
- modalState: { fundTransferOpen: boolean, limitOpen: boolean, confirmation: null | { type: string, details: object } }
Methods:
- handlePageChange(page)
- handleTransfer(details)
- handleLimitSave(newLimit)
- handleSnackbar(message, type)

HTML STRUCTURE (PSEUDO CODE ONLY)
<YouthAccountPage>
  <HeaderComponent user={user} onSearch={onSearch} />
  <SidebarComponent selectedPage={selectedPage} onNavigate={handlePageChange} />
  <MainContainer>
    {selectedPage === 'dashboard' && <DashboardPage ... />}
    {selectedPage === 'activity' && <ActivityPage ... />}
    {modalState.fundTransferOpen && <FundTransferModal ... />}
    {modalState.limitOpen && <SpendingLimitModal ... />}
    {modalState.confirmation && <ConfirmationModal ... />}
    <SnackbarComponent ... />
  </MainContainer>
</YouthAccountPage>

CSS SPECIFICATION
Layout:
- display: flex (main layout)
- Sidebar: fixed width, collapsible
- Header: sticky top
- MainContainer: flex-grow, responsive grid for cards
Spacing:
- 24px gap between sidebar/content
- 16px padding in content area
Component Styles:
- .youth-account-page
- .main-container
Responsive:
- Sidebar collapses to icons on < 900px
- Cards stack vertically on < 600px

API INTEGRATION
- On mount: GET /api/youth-accounts/{id}/dashboard, GET /transactions
- On transfer: POST /fund-transfer
- On limit save: PUT /spending-limit

ERROR HANDLING
- API failure: show SnackbarComponent with error
- Validation error: show inline or modal error

INTERACTION FLOW
- User selects nav → updates selectedPage
- User clicks Add Funds → opens FundTransferModal
- User submits transfer → API call → ConfirmationModal/Snackbar
- User sets limit → opens SpendingLimitModal → API call → ConfirmationModal/Snackbar

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- All error and confirmation states must use SnackbarComponent or ConfirmationModal
- All API errors must be surfaced to the user
- All modals must be accessible and responsive
