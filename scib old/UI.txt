1. Jira Requirement Summary

Story: Parent can allocate and manage funds for a youth account

Description:
As a parent or guardian, I want to allocate and manage funds for my child’s youth account so that I can teach financial responsibility while maintaining visibility and control over spending.

Acceptance Criteria:
- Scenario 1: Access youth account dashboard (dashboard shows current balance and recent activity)
- Scenario 2: Allocate funds to youth account (parent can transfer funds from their primary account)
- Scenario 3: Set spending limit (parent can configure a weekly spending limit, displayed on dashboard)
- Scenario 4: View youth spending activity (parent can see recent transactions)
- Scenario 5: Handle insufficient balance during fund allocation (error message shown)

UI Tasks (from subtasks):
- Design youth account dashboard layout
- Design fund transfer interaction for youth account
- Design spending limit configuration interface
- Design youth account activity view

5. Component Responsibilities

- YouthAccountDashboardPage: Orchestrates layout, data flow, and state for the dashboard.
- HeaderComponent: Displays logo, navigation tabs, search bar, and user profile.
- SidebarComponent: Provides navigation and quick actions for Add Funds/Set Limit.
- AccountOverviewCard: Shows youth name, avatar, balance, spending limit, and quick actions.
- RecentActivityCard: Displays a list/timeline of recent transactions.
- GuidanceCard: Shows tips and guidance for parents.
- Snackbar: Displays error/success messages (e.g., insufficient funds).
- Dialog/Modal: Handles fund transfer and spending limit configuration forms.


6. Data Flow / Props

YouthAccountDashboardPage
  -> HeaderComponent (userProfile, navigationTabs, onSearch)
  -> SidebarComponent (menuItems, onAddFunds, onSetLimit)
  -> AccountOverviewCard (youthName, avatarUrl, balance, spendingLimit, recentTransactionsCount, onAddFunds, onSetLimit)
  -> RecentActivityCard (transactions)
  -> GuidanceCard (tips)
  -> Snackbar (message, type)
  -> Dialog/Modal (open, type, onSubmit, onClose)

AccountOverviewCard
  -> CardHeader (youthName, avatarUrl)
  -> CardContent (balance, spendingLimit, recentTransactionsCount)
  -> CardFooter (onAddFunds, onSetLimit)

RecentActivityCard
  -> CardContent (transactions)

Dialog/Modal
  -> Form fields (amount, sourceAccount, limitValue, etc.)
  -> onSubmit, onClose
