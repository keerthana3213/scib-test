1. Jira Requirement Summary

Story: Parent can allocate and manage funds for a youth account

Description:
AS a parent or guardian I WANT to allocate and manage funds for my child’s youth account SO THAT I can teach financial responsibility while maintaining visibility and control over spending

Acceptance Criteria:
- Scenario 1: Access youth account dashboard (dashboard with current balance and recent activity)
- Scenario 2: Allocate funds to youth account (Add Funds, transfer from parent to youth)
- Scenario 3: Set spending limit (configure weekly spending limit, display on dashboard)
- Scenario 4: View youth spending activity (recent transactions)
- Scenario 5: Handle insufficient balance (error message on insufficient funds)

UI Tasks (from subtasks):
- Design youth account dashboard layout
- Design fund transfer interaction for youth account
- Design spending limit configuration interface
- Design youth account activity view

5. Component Responsibilities

HeaderComponent
- Displays application logo and navigation tabs (HOME, ASK, RESOURCES, REPORTS, ADMIN)
- Handles navigation events

TabsComponent
- Renders navigation tabs, highlights active tab

DashboardPromptCard
- Presents the main prompt ("Select ASK type")
- Contains option selection and action buttons

OptionButton
- Allows user to select "Core" or "Rotational" ASK type
- Triggers selection logic

ExitButton
- Allows user to exit the flow
- Styled as destructive (red)

StartNowButton
- Initiates the ASK process
- Disabled until an option is selected


6. Data Flow / Props

YouthAccountDashboardPage
 -> HeaderComponent (tabs, activeTabId, navigation handlers)
 -> DashboardPromptCard (promptText, optionButtons, actionButtons)

DashboardPromptCard
 -> OptionButtonsContainer (options: [Core, Rotational], selectedOption, onSelect)
 -> ActionButtonsContainer (onExit, onStartNow, startNowDisabled)

OptionButton
 -> label, variant, size, selected, onClick

ExitButton
 -> label="EXIT", variant="danger", onClick

StartNowButton
 -> label="START NOW", variant="secondary", disabled, onClick
