Structured UI Component Architecture

1. Jira Requirement Summary

Story Description:
AS a parent or guardian I WANT to allocate and manage funds for my child’s youth account SO THAT I can teach financial responsibility while maintaining visibility and control over spending.

Acceptance Criteria:
- Scenario 1: Access youth account dashboard (parent sees dashboard with current balance and recent activity)
- Scenario 2: Allocate funds to youth account (parent can transfer funds from primary to youth account)
- Scenario 3: Set spending limit (parent configures weekly spending limit, system saves/displays limit)
- Scenario 4: View youth spending activity (parent sees recent transactions)
- Scenario 5: Handle insufficient balance during fund allocation (error message shown)

UI Tasks/Subtasks:
- Design youth account dashboard layout
- Design fund transfer interaction for youth account
- Design spending limit configuration interface
- Design youth account activity view

4. UI Component Architecture

Screen: Youth Account Dashboard

YouthAccountDashboardPage
├── HeaderComponent (AavaHeaderComponent)
│   ├── TabsComponent (AavaTabsComponent) [for navigation: HOME, ASK, RESOURCES, REPORTS, ADMIN]
│   └── AvatarComponent (AavaAvatarsComponent) [user profile]
├── SidebarComponent (AavaSidebarComponent) [optional, for navigation]
├── MainContentContainer
│   ├── PromptBarComponent (AavaPromptBarComponent) ["Select ASK type"]
│   ├── OptionCardContainer (AavaGridComponent)
│   │   ├── OptionButtonComponent (AavaButtonComponent) [Core]
│   │   └── OptionButtonComponent (AavaButtonComponent) [Rotational]
│   ├── ActionButtonContainer (AavaGridComponent)
│   │   ├── ExitButtonComponent (AavaButtonComponent, variant="danger")
│   │   └── StartNowButtonComponent (AavaButtonComponent, variant="secondary", disabled)
│   └── SnackbarComponent (AavaSnackbarComponent) [for error messages]
└── DialogComponent (AavaDialogComponent) [for fund allocation, spending limit, or activity details]

5. Component Responsibilities

HeaderComponent
- Displays application logo, navigation tabs, and user avatar/profile.

TabsComponent
- Provides navigation between dashboard sections (HOME, ASK, etc).

SidebarComponent
- Optional: Provides additional navigation or quick links.

PromptBarComponent
- Displays the "Select ASK type" prompt.

OptionCardContainer
- Arranges option buttons (Core, Rotational) in a grid layout.

OptionButtonComponent
- Represents selectable ASK type options.

ActionButtonContainer
- Arranges action buttons (EXIT, START NOW) in a grid layout.

ExitButtonComponent
- Triggers exit action, styled as danger (red).

StartNowButtonComponent
- Triggers start action, styled as secondary, disabled until selection is made.

SnackbarComponent
- Displays error messages (e.g., insufficient funds).

DialogComponent
- Handles fund allocation, spending limit configuration, and activity details.

6. Data Flow / Props

YouthAccountDashboardPage
 -> HeaderComponent (userData, navigationTabs)
HeaderComponent
 -> TabsComponent (tabs: [HOME, ASK, RESOURCES, REPORTS, ADMIN])
 -> AvatarComponent (avatarUrl, userName)

MainContentContainer
 -> PromptBarComponent (promptText: "Select ASK type")
 -> OptionCardContainer (options: [Core, Rotational])
OptionCardContainer
 -> OptionButtonComponent (label, selected, onSelect)

ActionButtonContainer
 -> ExitButtonComponent (onExit)
 -> StartNowButtonComponent (onStart, disabled)

SnackbarComponent
 -> errorMessage (visible when error occurs)

DialogComponent
 -> fundAllocationData, spendingLimitData, activityDetails
