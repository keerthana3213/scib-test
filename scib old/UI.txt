----
1. Jira Requirement Summary

Story: Parent can allocate and manage funds for a youth account

Description:
AS a parent or guardian I WANT to allocate and manage funds for my child’s youth account SO THAT I can teach financial responsibility while maintaining visibility and control over spending

Acceptance Criteria:
- Scenario 1: Access youth account dashboard (parent sees dashboard with balance and activity)
- Scenario 2: Allocate funds to youth account (parent can transfer funds)
- Scenario 3: Set spending limit (parent configures weekly limit)
- Scenario 4: View youth spending activity (parent sees transactions)
- Scenario 5: Handle insufficient balance (error message shown)

Subtasks:
- Design youth account activity view
- Design spending limit configuration interface
- Design fund transfer interaction for youth account
- Design youth account dashboard layout

Linked Issues:
- Create API to retrieve youth account dashboard details
- Create API for parent to transfer funds to youth account
- Create API for configuring youth spending limit
- Create API to retrieve youth account transaction history
- Define OpenAPI/Swagger specification for youth account APIs

│   ├── ActionButtonContainer (Grid)

----
5. Component Responsibilities

HeaderComponent:
- Displays navigation bar with tabs
- Handles navigation state

NavigationTabsComponent:
- Renders tab options, highlights selected

SidebarComponent:
- Provides navigation/profile (optional, if dashboard expands)

TitleSectionComponent:
- Displays "Select ASK type" title

OptionCardContainer:
- Grid layout for option cards

OptionCardComponent:
- Displays option (Core/Rotational)
- Contains main action button

OptionButtonComponent:
- Triggers selection of ASK type
- Uses aava-button with variant and size props

ActionButtonContainer:
- Grid layout for action buttons

ExitButtonComponent:
- Exits the flow
- Uses aava-button, variant="danger", pill, prominent

StartNowButtonComponent:
- Starts the process
- Uses aava-button, variant="secondary", disabled

SnackbarComponent:
- Shows error messages (e.g., insufficient funds)

ModalComponent:
- Handles fund allocation, spending limit, transaction details
- Uses modal content projection (header, body, footer)

----
6. Data Flow / Props

YouthAccountDashboardPage
 -> NavigationTabsComponent (tabs, selectedTab)
 -> OptionCardContainer (options: [Core, Rotational])
    -> OptionCardComponent (optionType, isSelected)
       -> OptionButtonComponent (label, variant, size, onClick)
 -> ActionButtonContainer
    -> ExitButtonComponent (onClick)
    -> StartNowButtonComponent (disabled, onClick)
 -> SnackbarComponent (message, type)
 -> ModalComponent (modalType, data, onClose)
