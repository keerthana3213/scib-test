1. JIRA REQUIREMENT SUMMARY

Story Description:
Create Angular component for Kanban board container with three-column layout. Implement at src/app/features/kanban/components/kanban-board/kanban-board.component.ts. Use @Component decorator and selector 'app-kanban-board'. Template must have three column containers using CSS Grid (grid-template-columns: repeat(3, 1fr)). SCSS file must support responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). OnInit lifecycle hook initializes columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

6. COMPONENT DEFINITIONS

Component Name: HeaderComponent (AavaHeaderComponent)
Purpose:
- Application header with search, notifications, avatar
Props:
- logoUrl: string
- tabs: TabItem[]
- activeTabId: string
State:
- searchText: string
Events:
- onTabChange, onSearchChange, onGraphIconClick, onTranslateIconClick, onDayLightModeIconClick
API Binding:
- None (UI only)

Component Name: SidebarComponent (AavaSidebarComponent)
Purpose:
- Navigation for Kanban scenarios, branding, footer
Props:
- menuItems: MenuItem[]
- isCollapsed: boolean
State:
- isCollapsed: boolean
Events:
- collapseToggle
API Binding:
- None (UI only)

Component Name: KanbanBoardComponent
Purpose:
- Container for three Kanban columns, manages board state
Props:
- columns: KanbanColumn[]
- error: string | null
State:
- columns: [{id, title, cards: KanbanCard[]}]
- error: string | null
Events:
- onCardMove, onAddTask, onEditTask, onError
API Binding:
- GET /api/youth-accounts/{youthAccountId}/dashboard (for initial data)

Component Name: KanbanColumnComponent
Purpose:
- Represents a single Kanban column (To Do, In Progress, Done)
Props:
- column: {id, title, cards: KanbanCard[]}
- isLoading: boolean
State:
- cards: KanbanCard[]
Events:
- onAddCard, onMoveCard, onEditCard
API Binding:
- None (handled by parent)

Component Name: KanbanCardComponent (AavaDefaultCardComponent)
Purpose:
- Displays a single task card with title, description, meta
Props:
- card: {title, description, assignee, dueDate}
State:
- isSelected: boolean
Events:
- onSelect, onMove, onEdit
API Binding:
- None (handled by parent)

Component Name: InlineAddFormComponent / ModalComponent (AavaModal)
Purpose:
- Form for adding/editing a task (inline or modal)
Props:
- mode: 'add' | 'edit'
- card?: KanbanCard
State:
- form fields: title, description, assignee, dueDate
Events:
- onSubmit, onCancel
API Binding:
- POST /api/youth-accounts/{youthAccountId}/fund-transfer (if adding funds)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit (if editing limits)

Component Name: SnackbarComponent (AavaSnackbar)
Purpose:
- Shows error/success notifications
Props:
- message: string
- type: 'error' | 'success' | 'info'
State:
- isVisible: boolean
Events:
- onDismiss
API Binding:
- None

Component Name: DividerComponent (AavaDividers)
Purpose:
- Visual separation between columns/sections
Props:
- variant: 'solid' | 'dashed' | 'dotted' | 'gradient'
- orientation: 'horizontal' | 'vertical'
State:
- None
Events:
- None
API Binding:
- None

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch Kanban board data (columns, cards)
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Add funds (if mapped to add task)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update limits (if mapped to edit task/column)
- GET /api/youth-accounts/{youthAccountId}/transactions → List transactions (if mapped to card activity)

9. API TO UI MAPPING

Component → API:
- KanbanBoardComponent → GET /api/youth-accounts/{youthAccountId}/dashboard
- InlineAddFormComponent / ModalComponent → POST /api/youth-accounts/{youthAccountId}/fund-transfer (if adding funds)
- InlineAddFormComponent / ModalComponent → PUT /api/youth-accounts/{youthAccountId}/spending-limit (if editing limits)
- KanbanCardComponent → GET /api/youth-accounts/{youthAccountId}/transactions (for card activity)

13. INTERACTION FLOWS

User Action → UI → API
- Click "Add Task"
  → Open InlineAddFormComponent or ModalComponent
  → Submit
  → Call POST API (if mapped)
- Click "Edit Task"
  → Open InlineAddFormComponent or ModalComponent
  → Submit
  → Call PUT API (if mapped)
- Drag card to column
  → Update KanbanBoardComponent state
  → Optionally call API to persist
- Error in column config
  → Show SnackbarComponent or error-msg
- Click "Move" button
  → Move card between columns
  → Update state and optionally call API
