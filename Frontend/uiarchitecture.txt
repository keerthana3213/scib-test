## UI COMPONENT ARCHITECTURE

1. JIRA REQUIREMENT SUMMARY

Story Description:
Create Angular component for Kanban board container with three-column layout. Implement at src/app/features/kanban/components/kanban-board/kanban-board.component.ts. Use @Component decorator and selector 'app-kanban-board'. Template must have three column containers using CSS Grid (grid-template-columns: repeat(3, 1fr)). SCSS file must include responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). OnInit lifecycle hook initializes column data structure: columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', aria-labelledby for each column header. Implement error handling with *ngIf to display error message when column configuration fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport configuration.

Acceptance Criteria:
- Kanban board displays three columns: To Do, In Progress, Done
- Responsive layout for desktop, tablet, mobile
- ARIA attributes for accessibility
- Error message shown if column config fails
- Styling via Angular Material or custom CSS
- Responsive meta tags present

UI Tasks:
- Implement Kanban board container
- Define columns and cards structure
- Add ARIA/accessibility attributes
- Add error handling for config
- Responsive SCSS breakpoints
- Use grid layout

6. COMPONENT DEFINITIONS

Component Name: HeaderComponent
Purpose:
- Application header with search, notifications, and user avatar
Props:
- searchText: string
- user: User
State:
- activeTab: string
Events:
- onSearchChange
- onNotificationClick
API Binding:
- None

Component Name: SidebarComponent
Purpose:
- Navigation for Kanban scenarios, branding, footer
Props:
- menuItems: MenuItem[]
- selected: string
State:
- isCollapsed: boolean
Events:
- onMenuSelect
API Binding:
- None

Component Name: KanbanBoardComponent
Purpose:
- Container for Kanban columns, manages board state
Props:
- columns: KanbanColumn[]
State:
- error: string | null
Events:
- onAddTask(columnId)
- onCardMove(cardId, fromColumn, toColumn)
API Binding:
- None (delegates to child components)

Component Name: KanbanColumnComponent
Purpose:
- Represents a Kanban column (To Do, In Progress, Done)
Props:
- column: KanbanColumn
State:
- cards: KanbanCard[]
Events:
- onAddTask
- onCardDrop
API Binding:
- None

Component Name: KanbanCardComponent
Purpose:
- Displays a single Kanban card/task
Props:
- card: KanbanCard
State:
- isSelected: boolean
Events:
- onMove(toColumn)
- onEdit
API Binding:
- None

Component Name: AddTaskButtonComponent
Purpose:
- Button to trigger task creation modal
Props:
- columnId: string
Events:
- onClick
API Binding:
- None

Component Name: ModalComponent
Purpose:
- Modal overlay for task creation/edit
Props:
- isOpen: boolean
- mode: 'create' | 'edit'
- card?: KanbanCard
Events:
- onSubmit
- onCancel
API Binding:
- POST /api/youth-accounts/{youthAccountId}/tasks (if available)
- PUT /api/youth-accounts/{youthAccountId}/tasks/{taskId} (if available)

Component Name: SnackbarComponent
Purpose:
- Transient notification for errors/success
Props:
- message: string
- type: 'success' | 'error' | 'info'
- duration: number
Events:
- onDismiss
API Binding:
- None

Component Name: PopupComponent
Purpose:
- Confirmation, guided setup, support popups
Props:
- type: 'confirmation' | 'guided-setup' | 'error' | 'support'
- message: string
Events:
- onConfirm
- onCancel
API Binding:
- None

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch dashboard data
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Add funds
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update limit
- GET /api/youth-accounts/{youthAccountId}/transactions → List transactions

9. API TO UI MAPPING

Component → API:
- KanbanBoardComponent → GET /api/youth-accounts/{youthAccountId}/dashboard (for board data)
- ModalComponent (Task Creation) → (No direct API for tasks in OpenAPI, would require extension)
- ModalComponent (Task Edit) → (No direct API for tasks in OpenAPI, would require extension)
- KanbanBoardComponent → GET /api/youth-accounts/{youthAccountId}/transactions (if cards are mapped to transactions)

10. API COMPATIBILITY

Compatible:
- Dashboard API supports fetching board-level data
- Transactions API can be used for card/task history if mapped

Missing:
- No explicit API for Kanban task CRUD (create, update, delete)
- No API for Kanban column configuration

Transformations:
- If Kanban cards are mapped to transactions, transformation logic is needed to adapt transaction schema to card schema
- Recommend extending API with /tasks endpoints for full Kanban CRUD

13. INTERACTION FLOWS

User Action → UI → API
- Click "Add Task"
  → Open ModalComponent
  → Submit
  → (Would call POST /tasks if available)
- Drag Card
  → Move Card between columns
  → Update board state
  → (Would call PUT /tasks/{id} if available)
- Edit Card
  → Open ModalComponent in edit mode
  → Submit
  → (Would call PUT /tasks/{id} if available)
- Error (e.g., invalid drop)
  → Show SnackbarComponent with error message
- Guided Setup
  → Show PopupComponent for column setup

14. GAPS & RECOMMENDATIONS

Missing UI:
- ARIA-labelledby for column headers
- Explicit error state for column config failure (should be surfaced in UI)

Improvements:
- Add drag-and-drop support (HTML supports, Jira does not specify)
- Add inline edit for cards (HTML supports, Jira does not specify)
- Add support popups for user guidance

Performance:
- Use OnPush change detection for all feature components
- Lazy load Modal and Popup components
- Use memoized selectors for board state

API:
- Extend OpenAPI to include /tasks endpoints for Kanban CRUD
- Map Kanban cards to transactions only if business logic aligns

Accessibility:
- Ensure all ARIA attributes are present and correct
- Keyboard navigation for all interactive elements
