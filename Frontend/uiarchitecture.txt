## UI COMPONENT ARCHITECTURE

1. JIRA REQUIREMENT SUMMARY

Story Description:
Create Angular component for Kanban board container with three-column layout. Implement at src/app/features/kanban/components/kanban-board/kanban-board.component.ts. Use @Component decorator and selector 'app-kanban-board'. Template must have three column containers using CSS Grid (grid-template-columns: repeat(3, 1fr)). SCSS file must support responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px). OnInit lifecycle hook initializes columns = [{id: 'todo', title: 'To Do', cards: []}, {id: 'inprogress', title: 'In Progress', cards: []}, {id: 'done', title: 'Done', cards: []}]. Add ARIA attributes: role='region', aria-label='Kanban Board', aria-labelledby for each column header. Implement error handling with *ngIf to display error message if column config fails. Use Angular Material or custom CSS for styling. Add responsive meta tags and viewport config.

Acceptance Criteria:
- Kanban board with three columns: To Do, In Progress, Done
- Responsive layout for desktop, tablet, mobile
- ARIA accessibility attributes present
- Error message shown if column config fails
- Uses Angular Material or custom CSS
- Viewport and meta tags for responsiveness

UI Tasks:

6. COMPONENT DEFINITIONS

Component Name: HeaderComponent
Purpose:
- Displays top navigation bar with search, notifications, avatar
Props:
- user: object
- notifications: array
State:
- searchQuery: string
Events:
- onSearch(query)
- onNotificationClick()
API Binding:
- None

Component Name: SidebarComponent
Purpose:
- Displays navigation for Kanban scenarios
Props:
- scenarios: array
- selectedScenario: string
State:
- None
Events:
- onScenarioSelect(scenario)
API Binding:
- None

Component Name: KanbanBoardComponent
Purpose:
- Container for Kanban columns, manages board state
Props:
- columns: array (id, title, cards[])
- error: string
State:
- columns: array
- error: string
Events:
- onAddTask(columnId, task)
- onMoveCard(cardId, fromColumn, toColumn)
- onEditTask(cardId)
API Binding:
- GET /api/youth-accounts/{youthAccountId}/dashboard (for initial data)

Component Name: KanbanColumnComponent
Purpose:
- Renders a single Kanban column with header, cards, add form
Props:
- column: object (id, title, cards[])
- error: string
State:
- showAddForm: boolean
Events:
- onAddTask(task)
- onMoveCard(cardId, toColumn)
API Binding:
- None (delegated to parent)

Component Name: KanbanCardComponent
Purpose:
- Displays a single Kanban card (task)
Props:
- card: object (title, description, assignee, dueDate)
State:
- selected: boolean
Events:
- onEdit()
- onMove(toColumn)
API Binding:
- None (delegated to parent)

Component Name: AddTaskFormComponent
Purpose:
- Inline form to add a new task to a column
Props:
- columnId: string
State:
- title: string
- description: string
- assignee: string
- dueDate: string
Events:
- onSubmit(task)
- onCancel()
API Binding:
- POST /api/youth-accounts/{youthAccountId}/fund-transfer (if adding a fund transfer task)

Component Name: ModalComponent
Purpose:
- Generic modal for popups (welcome, confirmation, error, guided setup)
Props:
- visible: boolean
- title: string
- content: string | component
- actions: array
State:
- None
Events:
- onClose()
- onAction(action)
API Binding:
- None

Component Name: SnackbarComponent
Purpose:
- Displays transient messages (success, error)
Props:
- message: string
- type: string (success, error, info)
State:
- visible: boolean
Events:
- onClose()
API Binding:
- None

8. API SUMMARY

Available APIs:
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch Kanban board data (columns, cards)
- POST /api/youth-accounts/{youthAccountId}/fund-transfer → Add funds (could be mapped to add task if task is a fund transfer)
- PUT /api/youth-accounts/{youthAccountId}/spending-limit → Update spending limit (not directly used in Kanban, but could be for task types)
- GET /api/youth-accounts/{youthAccountId}/transactions → List transactions (could be mapped to activity/timeline)

9. API TO UI MAPPING

Component → API:
- KanbanBoardComponent → GET /api/youth-accounts/{youthAccountId}/dashboard
- AddTaskFormComponent → POST /api/youth-accounts/{youthAccountId}/fund-transfer (if task is a fund transfer)
- Timeline/Activity (if present) → GET /api/youth-accounts/{youthAccountId}/transactions

10. API COMPATIBILITY

Compatible:
- Dashboard API supports fetching Kanban board data (if mapped to columns/cards)
- Fund transfer API can be used for tasks involving fund movement
- Transactions API can be used for activity/timeline

Missing:
- No direct API for generic Kanban task CRUD (add/edit/delete non-fund tasks)
- No API for updating card status/column (move card)

Transformations:
- Map dashboard.cards/columns to Kanban columns/cards
- Use fund transfer API only for financial tasks; otherwise, require a Kanban-specific API

13. INTERACTION FLOWS

User Action → UI → API
- Click "Add Task" → Open AddTaskFormComponent → Submit → (If fund transfer) Call POST /fund-transfer
- Click "Move Card" → Update card status in KanbanBoardComponent → (No API, local state or custom endpoint)
- Click "Edit Task" → Open ModalComponent with AddTaskFormComponent in edit mode → Submit → (No API, local state or custom endpoint)
- Error in column config → Show SnackbarComponent or error-msg
- Responsive navigation → SidebarComponent updates selected scenario

14. GAPS & RECOMMENDATIONS

Missing UI:
- No explicit ARIA role='region' or aria-labelledby in HTML (should be added for accessibility)
- No UI for managing spending limits or viewing transactions (though available in API)

Improvements:
- Add ARIA attributes to all interactive regions and headers
- Implement drag-and-drop for cards (HTML shows draggable, Jira does not specify)
- Add Timeline/Activity section if using transactions API
- Add API endpoints for generic Kanban task CRUD (add/edit/delete/move)

Performance:
- Use OnPush change detection for all components
- Lazy load modals and popups
- Debounce search and form submissions
- Optimize rendering of large card lists
