HeaderComponent

### 4.3 KanbanBoardComponent

Component Type:
Feature

Mapped From A1:
KanbanBoardComponent

Purpose:
Container for Kanban columns, manages board state.

Parent:
KanbanBoardPage

Children:
- KanbanColumnComponent (x3)
- ModalComponent
- SnackbarComponent
- ErrorComponent

Dependencies:
- Grid (library)
- Modal (library)
- Snackbar (library)
- ErrorComponent

Library Components Used:
- Grid → Kanban columns layout
- Modal → task creation/edit
- Snackbar → notifications

TypeScript Specification
Inputs:
- columns: KanbanColumn[]
Outputs:
- onAddTask(columnId: string)
- onMoveCard(cardId: string, fromColumn: string, toColumn: string)
- onEditCard(cardId: string)
- onError(error: string)
State:
- columns: [{ id, title, cards }]
- error: string | null
Methods:
- fetchColumns()
- handleAddTask()
- handleMoveCard()
- handleEditCard()
- handleError()

HTML STRUCTURE (PSEUDO CODE ONLY)
<Grid columns="3">
  <KanbanColumnComponent *ngFor="let column of columns" [column]="column" />
</Grid>
<ModalComponent *ngIf="modalOpen" />
<SnackbarComponent *ngIf="snackbarOpen" />
<ErrorComponent *ngIf="error" [message]="error" />

CSS SPECIFICATION
Layout:
- display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px;
Spacing:
- margin: 24px 0;
Component Styles:
- .kanban-board, .kanban-column
Responsive:
- grid-template-columns: 1fr (mobile), 1fr 1fr (tablet), 1fr 1fr 1fr (desktop)

API INTEGRATION
- fetchColumns() → GET /api/youth-accounts/{youthAccountId}/dashboard
- handleAddTask() → POST to Kanban API (not present, extension needed)
- handleMoveCard() → PUT to Kanban API (not present, extension needed)

ERROR HANDLING
- If fetchColumns fails → set error, show ErrorComponent
- If add/move fails → show SnackbarComponent (type: error)

INTERACTION FLOW
- User clicks Add Task → open ModalComponent
- User submits modal → add task, refresh columns
- User drags card → move card, refresh columns

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- API for Kanban card CRUD is missing and must be extended
- ARIA attributes must be present for accessibility
- Responsive breakpoints: desktop (>1024px), tablet (768px–1024px), mobile (<768px)
- Error handling must show inline and snackbar messages
- Use OnPush change detection for performance
