### 4.3 KanbanBoardComponent

Component Type:
Feature

Mapped From A1:
KanbanBoardComponent

Purpose:
Container for three Kanban columns, manages board state and data.

Parent:
MainContainer

Children:
- KanbanColumnComponent (x3)
- SnackbarComponent
- DividerComponent (optional)

Dependencies:
- Grid (library)
- Divider (library)
- Snackbar (library)

Library Components Used:
- Grid → layout
- Divider → visual separation
- Snackbar → error/success

TypeScript Specification
Inputs:
- columns: KanbanColumn[]
- error: string | null
Outputs:
- onCardMove(cardId, fromColumnId, toColumnId)
- onAddTask(columnId, task)
- onEditTask(cardId, updates)
- onError(errorMsg)
State:
- columns: [{id, title, cards: KanbanCard[] }]
- error: string | null
Methods:
- fetchBoardData()
- handleCardMove()
- handleAddTask()
- handleEditTask()
- handleError()

HTML STRUCTURE (PSEUDO CODE ONLY)
<Grid columns={3}>
  <KanbanColumnComponent column={columns[0]} />
  <Divider orientation="vertical" />
  <KanbanColumnComponent column={columns[1]} />
  <Divider orientation="vertical" />
  <KanbanColumnComponent column={columns[2]} />
</Grid>
<SnackbarComponent message={error} type="error" visible={!!error} />

CSS SPECIFICATION
Layout:
- display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px;
Spacing:
- padding: 24px;
Component Styles:
- .kanban-board, .kanban-column, .kanban-divider
Responsive:
- @media (max-width: 1024px) { grid-template-columns: 1fr; }

API INTEGRATION
- fetchBoardData() → GET /api/youth-accounts/{youthAccountId}/dashboard
- handleAddTask() → POST /api/youth-accounts/{youthAccountId}/fund-transfer
- handleEditTask() → PUT /api/youth-accounts/{youthAccountId}/spending-limit

ERROR HANDLING
- If error, show SnackbarComponent with error message
- If columns config fails, show error message in board area

INTERACTION FLOW
- On mount → fetchBoardData()
- Card move → handleCardMove()
- Add/Edit task → handleAddTask()/handleEditTask()
- API error → handleError()

### 4.6 InlineAddFormComponent / ModalComponent (AavaModal)

Component Type:
Feature

Mapped From A1:
InlineAddFormComponent / ModalComponent

Purpose:
Form for adding/editing a task (inline or modal)

Parent:
KanbanColumnComponent

Children:
- Textbox (title)
- Textarea (description)
- Dropdown (assignee)
- Date-Picker (dueDate)
- Button (Save, Cancel)

Dependencies:
- Modal (library)
- Textbox (library)
- Textarea (library)
- Dropdown (library)
- Date-Picker (library)
- Button (library)

Library Components Used:
- Modal → container
- Textbox → title
- Textarea → description
- Dropdown → assignee
- Date-Picker → due date
- Button → Save, Cancel

TypeScript Specification
Inputs:
- mode: 'add' | 'edit'
- card?: KanbanCard
Outputs:
- onSubmit(task)
- onCancel()
State:
- form: {title, description, assignee, dueDate}
Methods:
- handleChange(field, value)
- handleSubmit()
- handleCancel()

HTML STRUCTURE (PSEUDO CODE ONLY)
<Modal visible={visible} title={mode === 'add' ? 'Add Task' : 'Edit Task'}>
  <Textbox label="Title" value={form.title} onChange={...} />
  <Textarea label="Description" value={form.description} onChange={...} />
  <Dropdown label="Assignee" options={assignees} value={form.assignee} onChange={...} />
  <DatePicker label="Due Date" value={form.dueDate} onChange={...} />
  <Button label="Save" onClick={handleSubmit} />
  <Button label="Cancel" onClick={handleCancel} />
</Modal>

CSS SPECIFICATION
Layout:
- display: flex; flex-direction: column;
Spacing:
- gap: 12px; padding: 24px;
Component Styles:
- .modal-form, .modal-form-field
Responsive:
- @media (max-width: 768px) { padding: 12px; }

API INTEGRATION
- handleSubmit() →
  - if mode === 'add': POST /api/youth-accounts/{youthAccountId}/fund-transfer
  - if mode === 'edit': PUT /api/youth-accounts/{youthAccountId}/spending-limit

ERROR HANDLING
- If validation fails, show inline error message
- If API fails, propagate error to parent

INTERACTION FLOW
- Fill form → Save → API call
- Cancel → close modal
