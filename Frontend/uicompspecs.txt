### 4.1 HeaderComponent

Component Type:
Layout

Mapped From A1:
HeaderComponent

Purpose:
Displays top navigation bar with search, notifications, and avatar

Parent:
KanbanBoardPage

Children:
- SearchBar (library)
- NotificationIcon (library/Icon)
- Avatar (library)

Dependencies:
- Search-Bar
- Icon
- Avatar

Library Components Used:
- Search-Bar → search input
- Icon → notification bell
- Avatar → user profile

TypeScript Specification
Inputs:
- user: User
- notifications: Notification[]
Outputs:
- onSearch(query: string)
- onNotificationClick()
State:
- searchQuery: string
Methods:
- handleSearch(query: string) → emits onSearch
- handleNotificationClick() → emits onNotificationClick

HTML STRUCTURE (PSEUDO CODE ONLY)
<Header>
  <Grid direction="row" align="center" justify="space-between">
    <AppBranding />
    <SearchBar [(ngModel)]="searchQuery" (search)="handleSearch($event)" />
    <Icon name="notifications" (click)="handleNotificationClick()" />
    <Avatar [user]="user" />
  </Grid>
</Header>

CSS SPECIFICATION
Layout:
- display: flex; align-items: center; justify-content: space-between;
Spacing:
- padding: 16px; gap: 16px;
Component Styles:
- .header { background: #fff; border-bottom: 1px solid #eee; }
Responsive:
- @media (max-width: 768px) { flex-direction: column; }

API INTEGRATION
- None (UI only)

ERROR HANDLING
- None (UI only)

INTERACTION FLOW
User types in search → handleSearch → emits onSearch
User clicks notification → handleNotificationClick → emits onNotificationClick

### 4.3 KanbanBoardContainer

Component Type:
Feature

Mapped From A1:
KanbanBoardContainer

Purpose:
Main Kanban board grid with three columns

Parent:
KanbanBoardPage

Children:
- KanbanColumnComponent (To Do)
- KanbanColumnComponent (In Progress)
- KanbanColumnComponent (Done)
- ErrorSnackbar (conditional)

Dependencies:
- Grid
- KanbanColumnComponent
- Snackbar

Library Components Used:
- Grid → layout columns
- Snackbar → error display

TypeScript Specification
Inputs:
- columns: KanbanColumn[]
- error: string
Outputs:
- onAddTask(columnId: string)
- onCardMove(cardId: string, fromColumn: string, toColumn: string)
State:
- columns: KanbanColumn[]
- error: string
Methods:
- handleAddTask(columnId: string)
- handleCardMove(cardId: string, fromColumn: string, toColumn: string)

HTML STRUCTURE (PSEUDO CODE ONLY)
<KanbanBoard role="region" aria-label="Kanban Board">
  <Grid columns="3" gap="24px">
    <KanbanColumnComponent *ngFor="let col of columns" [column]="col" (addTask)="handleAddTask(col.id)" (cardMove)="handleCardMove($event.cardId, $event.from, $event.to)" />
  </Grid>
  <Snackbar *ngIf="error" [message]="error" />
</KanbanBoard>

CSS SPECIFICATION
Layout:
- display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px;
Spacing:
- padding: 24px;
Component Styles:
- .kanban-board { background: #fafbfc; }
Responsive:
- @media (max-width: 1024px) { grid-template-columns: 1fr; }

API INTEGRATION
- ngOnInit() → GET /api/youth-accounts/{youthAccountId}/dashboard → fetch columns/cards

ERROR HANDLING
- If error, show Snackbar with error message

INTERACTION FLOW
User clicks add task → handleAddTask → open AddTaskModal
User drags card → handleCardMove → update state (optionally call API)

- ariaLabel: string
Outputs:

- handleCardDrop(cardId: string)

HTML STRUCTURE (PSEUDO CODE ONLY)

  <header id="column-header-{{column.id}}">

    <Chip-Tag *ngIf="column.summary" [label]="column.summary" />

    <Button icon="add" (click)="handleAddTask()">Add</Button>

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- ARIA attributes must be present for accessibility
- Responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px)
- Error handling must use Snackbar
- All API calls must handle errors and show error messages
