### 4.1 HeaderComponent (AavaHeaderComponent)

Component Type:
Layout

Mapped From A1:
HeaderComponent

Purpose:
Application header with search, notifications, and user avatar.

Parent:
KanbanBoardPage

Children:

- NotificationIcon

- Icon

Methods:

- handleSearchChange(text)
- handleIconClick(icon)

  <Logo src={logoUrl} />

  <SearchBar value={searchText} onChange={searchChange} />
  <IconButton icon="notifications" onClick={iconClick} />

</Header>

CSS SPECIFICATION

Layout:
- display: flex; align-items: center; justify-content: space-between;

Spacing:

Component Styles:

- Avatar: size 32px

Responsive:

API INTEGRATION

ERROR HANDLING
- None required

INTERACTION FLOW
- Tab click → tabChange event
- Search input → searchChange event

### 4.3 KanbanBoardComponent

Component Type:
Feature

Mapped From A1:
KanbanBoardComponent

Purpose:
Container for Kanban columns, manages board state.

Parent:
KanbanBoardPage

Children:
- KanbanColumnComponent (x3)
- ModalComponent
- SnackbarComponent
- ErrorComponent

Dependencies:
- Modal
- Snackbar
- ErrorComponent

Library Components Used:
- Modal (AavaModalComponent) → Task creation/edit
- Snackbar (AavaSnackbarComponent) → Notifications

TypeScript Specification

Inputs:
- columns: KanbanColumn[]

Outputs:
- onAddTask(columnId: string)
- onMoveCard(cardId: string, fromColumn: string, toColumn: string)

State:
- columns: KanbanColumn[]
- error: string | null
- modalOpen: boolean
- selectedTask: KanbanCard | null

Methods:
- handleAddTask(columnId)
- handleMoveCard(cardId, from, to)
- handleModalSubmit(task)
- handleModalCancel()
- handleErrorClose()

HTML STRUCTURE (PSEUDO CODE ONLY)

<KanbanBoard role="region" aria-label="Kanban Board">
  <KanbanColumnComponent column={columns[0]} />
  <KanbanColumnComponent column={columns[1]} />
  <KanbanColumnComponent column={columns[2]} />
  <ModalComponent open={modalOpen} task={selectedTask} onSubmit={handleModalSubmit} onCancel={handleModalCancel} />
  <SnackbarComponent message={snackbarMessage} type={snackbarType} onClose={handleSnackbarClose} />
  <ErrorComponent message={error} onClose={handleErrorClose} />
</KanbanBoard>

CSS SPECIFICATION

Layout:
- display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px;

Spacing:
- padding: 24px;

Component Styles:
- KanbanBoard: background, border-radius

Responsive:
- @media (max-width: 1024px): grid-template-columns: 1fr 1fr
- @media (max-width: 768px): grid-template-columns: 1fr

API INTEGRATION
- GET /api/youth-accounts/{youthAccountId}/dashboard → Fetch board state
- (If API exists) POST/PUT for task actions

ERROR HANDLING
- If columns config fails, set error and show ErrorComponent
- API errors → show SnackbarComponent

INTERACTION FLOW
- Add Task → open ModalComponent
- Move Card → update state, call API if available
- Error → show ErrorComponent

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- CSS should include only key classes and layout rules, not full stylesheets
- ARIA attributes must be present for accessibility
- Responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px)
- Error handling must show inline or snackbar messages
- API integration is limited to available endpoints; Kanban CRUD requires backend extension
