### 4.1 HeaderComponent

HeaderComponent

KanbanBoardPage

- Avatar (aava-avatar) → user image

- Button (aava-button) → actions

API INTEGRATION
- None

ERROR HANDLING
- None

INTERACTION FLOW

### 4.3 KanbanBoardComponent

Component Type:
Feature

Mapped From A1:
KanbanBoardComponent

Purpose:
Container for Kanban columns, manages board state and error handling.

Parent:
KanbanBoardPage

Children:
- KanbanColumnComponent (x3)
- SnackbarComponent (conditional)

Dependencies:
- aava-grid
- aava-snackbar

Library Components Used:
- Grid (aava-grid) → columns layout
- Snackbar (aava-snackbar) → error messages

TypeScript Specification
Inputs:
- columns: KanbanColumn[]
- error: string

Outputs:
- onAddTask(columnId: string, task: Task)
- onMoveCard(cardId: string, fromColumn: string, toColumn: string)
- onEditTask(cardId: string)

State:
- columns: KanbanColumn[]
- error: string

Methods:
- fetchBoardData() → GET /api/youth-accounts/{youthAccountId}/dashboard
- handleAddTask(columnId, task)
- handleMoveCard(cardId, fromColumn, toColumn)
- handleEditTask(cardId)

HTML STRUCTURE (PSEUDO CODE ONLY)
<aava-grid class="kanban-board" grid-template-columns="repeat(3, 1fr)" role="region" aria-label="Kanban Board">
  <KanbanColumnComponent *ngFor="let column of columns" [column]="column" (addTask)="handleAddTask(column.id, $event)" (moveCard)="handleMoveCard($event.cardId, column.id, $event.toColumn)" (editTask)="handleEditTask($event.cardId)" />
  <aava-snackbar *ngIf="error" [message]="error" type="error" />
</aava-grid>

CSS SPECIFICATION
Layout:
- display: grid; grid-template-columns: repeat(3, 1fr); gap: 24px;
Spacing:
- padding: 24px;
Component Styles:
- .kanban-board { min-height: 60vh; }
Responsive:
- @media (max-width: 1024px) { grid-template-columns: 1fr; }
- @media (min-width: 768px) and (max-width: 1024px) { grid-template-columns: repeat(2, 1fr); }

API INTEGRATION
- fetchBoardData() → GET /api/youth-accounts/{youthAccountId}/dashboard

ERROR HANDLING
- If error, show snackbar with error message

INTERACTION FLOW
- On mount, fetchBoardData
- User adds/moves/edits card → emits events to parent

### 4.4 KanbanColumnComponent

Component Type:
Feature

Mapped From A1:
KanbanColumnComponent

Purpose:
Renders a single Kanban column with header, cards, and add form.

Parent:
KanbanBoardComponent

Children:
- AddTaskFormComponent
- KanbanCardComponent (n)

Dependencies:
- aava-card
- aava-button
- aava-divider

Library Components Used:
- Card (aava-card) → column container
- Button (aava-button) → add task
- Divider (aava-divider) → separation

TypeScript Specification
Inputs:
- column: KanbanColumn
- error: string

Outputs:
- onAddTask(task: Task)
- onMoveCard(cardId: string, toColumn: string)

State:
- showAddForm: boolean

Methods:
- toggleAddForm()
- handleAddTask(task)
- handleMoveCard(cardId, toColumn)

HTML STRUCTURE (PSEUDO CODE ONLY)
<aava-card class="kanban-column" role="region" [attr.aria-labelledby]="'column-' + column.id">
  <div class="column-header" id="column-{{column.id}}">
    <span class="column-title">{{column.title}}</span>
    <span class="column-summary">{{column.cards.length}} tasks</span>
    <span class="column-timestamp">Last updated: {{column.updatedAt | date}}</span>
    <aava-button (click)="toggleAddForm()">Add Task</aava-button>
  </div>
  <aava-divider />
  <AddTaskFormComponent *ngIf="showAddForm" [columnId]="column.id" (submit)="handleAddTask($event)" (cancel)="toggleAddForm()" />
  <KanbanCardComponent *ngFor="let card of column.cards" [card]="card" (move)="handleMoveCard($event.cardId, $event.toColumn)" (edit)="handleEditTask($event.cardId)" />
  <div *ngIf="error" class="error-msg">{{error}}</div>
</aava-card>

CSS SPECIFICATION
Layout:
- display: flex; flex-direction: column; gap: 16px;
Spacing:
- padding: 16px;
Component Styles:
- .column-header { display: flex; align-items: center; justify-content: space-between; gap: 8px; }
- .error-msg { color: #d32f2f; margin-top: 8px; }
Responsive:
- @media (max-width: 768px) { .kanban-column { min-width: 100%; } }

API INTEGRATION
- None (delegated to parent)

ERROR HANDLING
- If error, show .error-msg

INTERACTION FLOW
- User clicks Add Task → show AddTaskFormComponent
- User submits/cancels form → hide form
- User moves/edits card → emits events

8. NOTES & CONSTRAINTS

- HTML must follow A1 structure strictly
- Do NOT create new components unnecessarily
- Prefer reusable library components
- HTML must be pseudo-code ONLY (NOT real framework code)
- ARIA attributes must be present on Kanban board and columns
- Responsive breakpoints: desktop (>1024px), tablet (768px-1024px), mobile (<768px)
- Error handling: show snackbar or inline error message
- Use OnPush change detection for all components
- Lazy load modals and popups
- Debounce search and form submissions
- Optimize rendering of large card lists
